# airdb.space — 星辰大海

> 我们的征途是星辰大海。
> To the stars, and into the deep.

airdb.space 是一个关于「探索」的官方网站：向上仰望星空（航空航天、深空探测），向下潜入深海（深海科技、海洋探索），而支撑这一切的，是我们在 AI 与工程技术上的长期积累。它不是一个产品页，而是一个愿景的载体 —— 展示人类共同追求的方向，以及我们朝这个方向走出的每一步。

---

## 1. 定位与愿景

**一句话定位**：一个有科技感、未来感、前沿范的愿景型官网，讲述「星辰大海」这个人类共同的方向。

**三层叙事结构**：

| 层次 | 主题 | 内容锚点 |
|------|------|----------|
| 仰望 · 星辰 | 航空航天 / 深空探测 | 对标 SpaceX、Blue Origin；关注中国航天：嫦娥探月工程、天宫空间站、载人航天、行星探测（天问）等 |
| 潜入 · 大海 | 深海科技 / 海洋探索 | 深潜器（奋斗者号等）、深海观测、海洋工程 —— 星辰大海，一半在海 |
| 根基 · 科技 | AI 与工程积累 | 我们为什么有能力谈探索：AI、数据、基础设施等方面的长期技术沉淀（airdb 开源生态） |

**基调**：宏大但克制。愿景可以远，但每一步都要走得稳 —— 网站本身的迭代节奏也遵循这个原则（见第 5 节路线图）。

## 2. 站点结构（Site Map）

```
/                    首页：全屏星空 Hero + 三层叙事导览
├── /space           星辰 —— 航天专题：时间线（人类航天里程碑 + 中国航天大事记）
├── /ocean           大海 —— 深海专题：深度轴（从海平面到万米深渊的滚动叙事）
├── /technology      科技 —— 我们的积累：AI / 开源项目 / 工程能力
├── /journey         历程 —— 我们自己的发展时间线，每一步怎么走稳的
├── /en/*            英文版 —— 以上全部页面的 English 版本（hreflang 互链）
└── /about           关于 —— 愿景陈述 + 联系方式（info@airdb.com）
```

**首页信息流**（自上而下）：

1. **Hero**：全屏深空背景（星野 + 微弱视差），一句 Slogan：「我们的征途是星辰大海」，中英双语
2. **三扇门**：星辰 / 大海 / 科技，三个入口卡片，各自一句引导语
3. **探索时间线精选**：横向滚动，混排人类探索里程碑（阿波罗、旅行者、嫦娥、奋斗者号……）
4. **科技根基**：为什么是我们 —— AI 与工程积累的简短陈述
5. **Footer**：愿景一句话收尾 + 链接（GitHub: github.com/airdb 等）

## 3. 视觉与设计语言

**关键词**：深空、深海、克制的科技感。避免廉价的「赛博风」堆砌，向 SpaceX / Blue Origin 官网的气质看齐 —— 大面积留白（留黑）、大图、少文字、强排版。

- **色彩**：以近黑的深空蓝（`#050810` 一类）为底；星辰篇章用冷白与星光银，大海篇章渐入深海蓝绿；点缀色用一个克制的高亮色（如离子蓝 / 引擎橙），全站不超过两个强调色
- **字体**：中文用现代无衬线（思源黑体 / 系统字体栈），英文/数字用带工程感的字体，大标题字重轻、字号大、字距宽
- **动效**：滚动驱动的渐显与视差为主，克制使用；星空粒子背景要轻量（Canvas，尊重 `prefers-reduced-motion`）；「深海页」用滚动下潜的深度计作为叙事线索
- **图像**：优先使用 NASA / 公有领域的真实影像（真实感 > 插画感），统一做深色调色处理

## 4. 技术方案

| 项 | 选型 | 理由 |
|----|------|------|
| 框架 | **Astro**（静态优先，Islands 按需交互） | 内容型官网首选：零 JS 起步、性能极佳、SEO 友好 |
| 包管理 | pnpm | 已定 |
| 构建/任务 | Makefile（`make install / run / build`） | 已定，保持现状 |
| 样式 | Tailwind CSS + 少量自定义 CSS（动效） | 快速迭代 + 设计一致性 |
| 内容 | Astro Content Collections（Markdown/MDX） | 时间线条目、专题文章都以内容驱动，便于持续更新 |
| 交互点缀 | 原生 Canvas 星空 / 滚动动效（IntersectionObserver） | 不引入重型 3D 依赖，保证加载速度 |
| 部署 | 静态托管（Cloudflare Pages / Vercel / GitHub Pages 任一） | 待定 |

**性能与体验底线**：Lighthouse 90+；首屏无阻塞大依赖；移动端完整可用；深色为默认且唯一主题（与深空基调一致）。

## 5. 路线图 —— 每一步都走稳

- **Phase 1 · 点火** ✅：搭好 Astro 骨架 + 首页（Hero、三扇门、时间线、科技根基、Footer），先让愿景立起来
- **Phase 2 · 入轨** ✅：星辰页 `/space`（探索时间线）+ 科技页 `/technology`
- **Phase 3 · 巡航** ✅：大海页 `/ocean`（滚动下潜叙事）+ 历程页 `/journey`
- **Phase 4 · 深空** ✅：细节打磨 —— 动效（流星）、多语言（中/英 + hreflang）、SEO（sitemap / robots / canonical）、分享卡片（OG image）

每个 Phase 独立可上线，做完一步、站稳一步，再走下一步。
